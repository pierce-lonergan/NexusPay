# Journal — 2026-06-10

## Session 1 — B-011 + full integration-test bring-up → first-ever green CI

### Arc
Started to "implement B-011 (Flyway version collisions)" with CI as the verifier
(L2). It turned out B-011 was the single cork in the bottle: the V1×4/V2×2
collision had aborted the schema before any of the 13 integration tests could ever
boot the app context — so ~10 layers of latent drift had accumulated unverified.
Clearing it made CI surface each layer in turn. Fixed every one with ground-truth
from the CI `test-results` artifact (the Gradle console hides exception messages;
the JUnit HTML captures the full Flyway log + SQL + stack).

### Fixes, in the order CI surfaced them (each its own commit, CI-verified)
1. Flyway resolver collision → renumbered ledger/iam/gateway/payment V1/V2 into a
   unique pre-V2001 band; deleted the 13 customizers (WRONG — see below).
2. Discovery: deleting the customizers silently dropped 11 modules (bare
   `db/migration` recursion is unreliable across JARs). → explicit per-module LEAF
   locations + `fail-on-missing-locations`; moved app's loose migrations to a leaf.
   THEN the real one: the TEST yaml pinned a 4-leaf list that REPLACED the base and
   dropped the `app` leaf (current_tenant_id) → made it inherit the full list.
3. V3015 / V4002 / V4004 RLS policies referenced a tenant_id column the child table
   doesn't have (split_rules, webhook_triggers) → parent-derived subquery.
4. Hibernate validate cascade: created Modulith event_publication (+event_type);
   @JdbcTypeCode(JSON) on 23 jsonb String columns; List/array→jsonb (events);
   Double vs DECIMAL. Used javap on the Modulith jar + a static entity↔migration
   scanner to BATCH classes instead of one-CI-cycle-each.
5. Wiring: @EnableJpaRepositories(considerNestedRepositories=true); @Qualifier on
   the duplicate Kafka ConsumerFactory beans.
6. Runtime: tenant SQL `SET … = ?` is illegal → set_config(); disabled Vault in
   tests; made PSP/Keycloak health indicators @ConditionalOnEnabledHealthIndicator.
7. Assertions: AccessDeniedException → 403 not 500 (missing @ExceptionHandler).
→ **CI GREEN: 263 tests, all 13 integration tests pass, app boots on real infra.**
